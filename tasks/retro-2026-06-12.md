## What Went Well
- **Clean separation of concerns** in the backend: spawn mechanism (`claude-runner.js`),
  validation (`validate.js`), config (`config.js`), prompt assembly (`prompt.js`),
  persistence (`db.js`), and HTTP (`server.js`) are each isolated and small.
- **Security was designed in, not bolted on.** Three independent defenses landed in
  the same pass: argv-not-shell prompt passing, slug charset enforcement + path-escape
  re-check, and `ANTHROPIC_API_KEY` stripped from the child env.
- **Verified headless mechanism**: `claude -p <prompt> --output-format stream-json
  --verbose --model <m> --permission-mode acceptEdits --allowedTools ... --add-dir .`
  with line-delimited JSON parsing and `session_id` capture for `--resume` iteration.
- **Zero external API spend** by construction — subscription OAuth only, enforced by
  env stripping rather than convention.

## Patterns
- **Defense-in-depth on every untrusted boundary**: `toSlug()` produces a safe slug,
  then `assertSafeSlug()` re-validates it, then `projectDir()` confirms the resolved
  path stays under `GENERATED_ROOT`. Same value validated three times at three layers.
- **Spawn child processes with discrete argv arrays, never shell-interpolated strings** —
  the single most reusable instinct from this build for any CLI-wrapping backend.
- **Allowlist + denylist tool scoping** for spawned agents: explicit `--allowedTools`
  (scoped file ops) and `--disallowedTools` (rm/git/curl/npm/node/web) together.
